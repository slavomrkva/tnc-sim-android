## What this is
The **Android app wrapper** for TNC Sim (a Heidenhain TNC Klartext CNC
simulator), built with **Capacitor**. It packages the same web app as a native
Android app so it runs fully offline with no browser chrome/address bar,
published on the **Play Store** as `org.tncsim.twa`.

This app previously shipped as a **TWA (Trusted Web Activity)** built via
PWABuilder. It was replaced by this Capacitor build as a Play Store *update*
to the same package — see rule #1, this is not a fresh app listing.

## Relationship to the web repo — READ THIS
The actual product source of truth is **`slavomrkva/tnc-sim`** (a separate
GitHub repo, the live site at **tncsim.org**). `www/index.html` here is a
**copy** of that repo's `index.html`, synced in manually. **Do not edit
`www/index.html` (or the root-level `index.html` left over from history)
directly with feature changes** — edit the source in `tnc-sim`, push it there,
then sync a fresh copy here (see "Update flow" below). The one exception is
app-specific behaviour gated behind `window.Capacitor` (e.g. hiding the web
header, an app-only onboarding) — that also belongs in `tnc-sim`'s
`index.html` since it's the same file, just conditionally branching on
`window.Capacitor`.

## Files in the repo
- `android/` — the native Android/Gradle project (generated by
  `npx cap add android`, then customized).
- `www/index.html` — the web app copy that gets bundled into the app on
  `npx cap sync android`.
- `capacitor.config.json` — Capacitor config (`appId: org.tncsim.twa`).
- `package.json` / `package-lock.json` — just the Capacitor CLI/core/android
  deps. No web build tooling — the web app itself has no build step.
- `NOTES.md` — this file.
- Root-level `index.html`, `manifest.json`, `service-worker.js`, etc. — legacy
  leftovers from when this folder was cloned from `tnc-sim`. Not used by the
  Android build (only `www/index.html` is). Harmless but don't rely on them
  being current.

## No build / no tooling (for the web content)
The web app itself is plain vanilla JS, no framework, no bundler — same rule
as the `tnc-sim` repo. The only "build" here is the Android/Gradle build.

## Update flow (how to ship a new Android release)
1. Make the actual product change in `tnc-sim` (the web repo), commit, push —
   confirm it's live on tncsim.org.
2. Here: `git pull`, then copy the current `tnc-sim/index.html` into
   `www/index.html` **verbatim, no edits** (unless the change IS an
   app-specific `window.Capacitor` branch, which still originates in
   `tnc-sim`).
3. `npm install` (first time / after dependency changes), then
   `npx cap sync android`.
4. Bump `versionCode` (integer, always +1 from the last Play Store release)
   and `versionName` in `android/app/build.gradle`. **Never reuse or lower a
   versionCode** — Play Store will reject the upload.
5. Commit + push here.
6. On the dev machine (this repo has no access to the signing keystore or
   Android Studio): `git pull`, open in Android Studio, **Generate Signed
   Bundle** using the *original* keystore (the one used for the very first
   Play Store upload — see rule #2), produce the `.aab`.
7. Upload the `.aab` to Play Console as a new release on the existing
   `org.tncsim.twa` listing.

## NON-OBVIOUS RULES — do not break these
<!-- Add a new numbered rule here whenever a new critical pitfall is found. -->

### 1. `applicationId` must stay `org.tncsim.twa` forever
This app replaces a prior TWA release under the same package name, as a Play
Store *update*, not a new listing. Changing `applicationId` in
`android/app/build.gradle` (or `capacitor.config.json`'s `appId`) would make
Play Store treat it as a brand-new app — existing installs/reviews/listing
would not carry over. Never change it without explicit instruction.

### 2. Must sign every release with the SAME keystore
Play Store rejects an update signed with a different key than the app's
existing signing key. The keystore file and its password are **never
committed to this repo** (see `.gitignore`: `*.keystore`,
`signing-key-info.txt`) — they only exist locally on the dev machine and must
be reused for every future release. Losing that keystore means losing the
ability to update the app at all.

### 3. NEVER push this project's `origin` at the web repo (`tnc-sim`)
This repo's local clone was originally created by copying/cloning the
`tnc-sim` working directory (that's why old git history is shared), and its
`origin` remote defaulted to `tnc-sim.git`. Before pushing anything from this
project, always double-check with `git remote -v` that `origin` points to
**`tnc-sim-android`**, not `tnc-sim`. This already happened once: the entire
Android project (68 files, including `android/`, `www/`, `package.json`) got
pushed straight into `tnc-sim` main, which risked breaking the Cloudflare
Pages static-site deploy (a `package.json` at repo root can make some static
hosts try to run a build instead of serving files as-is). It had to be
reverted. If you're an AI assistant working across both repos in one session,
verify the target repo before every push, no exceptions.

### 4. Never commit secrets — check before every commit
This project's working directory has, at various points, contained a GitHub
2FA recovery codes file (`github-recovery-codes.txt`), local Windows junk
(`desktop.ini`), and stray unrelated files (PDFs, "New Text Document.txt")
that briefly ended up staged for commit. `.gitignore` now excludes
`node_modules/`, Gradle build output, `android/local.properties`,
`*.keystore`, `signing-key-info.txt`, `github-recovery-codes.txt`, and
`desktop.ini` — but `.gitignore` only stops *future* additions, it does not
retroactively scrub files already committed. Before any push, run
`git show --stat HEAD` (or `git status`) and actually look at the file list —
don't just trust `.gitignore`. If a secret was ever committed even locally
and not yet pushed, prefer `git commit --amend` (safe pre-push) over trying to
just delete-and-recommit (leaves it in history).

### 5. `www/index.html` is a synced copy, not a fork
Don't hand-edit `www/index.html` to add Android-only tweaks unless they're
gated behind `window.Capacitor` (or similar) so the *same* source file still
works correctly as the plain web app. If it diverges silently from
`tnc-sim/index.html`, the next sync will either clobber your Android-only
tweak or you'll forget it exists and debug the wrong file.

## Testing checklist before shipping a release
- `www/index.html` matches `tnc-sim`'s current `index.html` (diff them) unless
  intentionally different for an app-specific reason.
- `versionCode` is strictly higher than the last Play Store release.
- `applicationId` is still `org.tncsim.twa`.
- `git remote -v` on `origin` points to `tnc-sim-android`, not `tnc-sim`.
- `git status` / `git show --stat HEAD` shows no keystore, recovery codes, or
  unrelated personal files before pushing.
- Signed with the original keystore (ask the human if unsure — don't guess).
